## Power BI Desktop Tasks

These can only be done by editing `Energy Co Pilot.pbix` in Power BI Desktop
and re-publishing to the workspace. The web app cannot reach into the embed
iframe to style PBI visuals — they're cross-origin.

### 1. Visual Borders Across All 9 Pages

**Why:** Per-tile borders make the dashboard look professionally framed
inside the embed wrap. Mert specifically asked for "her görsele ayrı ince
çerçeve" (a thin frame around every visual).

**Steps:**
1. Open `Energy Co Pilot.pbix` in Power BI Desktop
2. Go to each of the 9 pages (01_portfolio_overview ... 09_Battery_Strategy)
3. Select all visuals on the page (`Ctrl+A` in the report canvas)
4. Open the **Format** pane → **General** → **Effects**
5. Toggle **Visual border** ON
   * Color: `#1D9E75` (brand emerald) — or per-page accent if you want
     each page to feel distinct
   * Width: 1 px
   * Rounded corners: 4-6 px
6. (Optional) Visual background ON with a very low transparency (e.g. 5%)
   to make the tile lift slightly off the canvas
7. Publish: **Home** → **Publish** → workspace `Energy-Copilot-Platform`

**Validation:** Open `localhost:3000/buildings/B003` after publishing.
Each KPI card, slicer, and chart should now have its own emerald border.

### 2. "Hedef" → "Goal" Localisation

**Why:** Several KPI subtitles render `"Hedef: 45 (%+100)"` — Turkish for
"target". The product is English-only by policy ([[feedback_language]]).

**Steps:**
1. Open `.pbix` in Desktop
2. Open the **Model view** → **Measures**
3. Search for DAX expressions containing the literal `"Hedef: "`. Likely
   suspects: `_pf_target_display`, `_kpi_target_label`, page-level
   subtitle measures
4. Replace `"Hedef: "` → `"Goal: "` in every matching measure
5. Save + publish

### 3. Per-Page Background Refactor (Stretch)

**Why:** Each PBI page currently has a 1920×1080 full-bleed background
image. The web app's embed wrap clips this. We get a cleaner look if PBI
pages use **transparent** backgrounds + the web app chrome provides all
the branding (logo, motifs, hero band).

**Steps:**
1. For each page: **Format** → **Page background** → set to transparent
2. Remove any background images that were full-bleed
3. Verify visuals still readable on the dark navy web app background
4. Publish

**Trade-off:** PBI Service direct viewing loses the branded look. If you
ever embed the report in another product, you'll want to provide branding
from that product's chrome instead.

### 4. Building Name Slicer Visibility

**Why:** On `/buildings/B003`, the Building Name slicer shows only one
option ("Hamburg Logistics Hub Gamma") because the embed-time filter
already locked the building. The slicer becomes redundant UI.

**Options:**
a. **Hide** the slicer on per-building view → not possible from web app,
   would need a separate report variant in PBI
b. **Leave it** — it's harmless and consistent across pages. Pick this
   unless it actively confuses users.
