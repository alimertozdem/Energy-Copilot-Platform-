### РЉа PBI embed 400 Рђћ root cause + fix
A DirectLake dataset that defines an RLS role (`CustomerRLS`) **requires** an
`effectiveIdentity` in the V2 embed token when embedded via a service principal.
The previous admin path **omitted** the identity Рєњ Power BI rejected `GenerateToken`
with 400. Fix: the admin now also gets a `CustomerRLS` identity, but with
`customData` = **every** `building_id` in the model (`SELECT DISTINCT building_id
FROM silver_building_master`), so `PATHCONTAINS` matches every row Рєњ admin sees the
whole model, identity valid. Non-admins unchanged (own buildings, empty = fail-closed).

### РЉБ Per-dwelling COРѓѓ Рђћ honesty note

heating-COРѓѓ cost by each dwelling's **living-area share (Wohnfl├цche pro-rata)**. This

### B) Notebook 06 Рђћ operational measures real estimate (Fabric, Mert runs)
**Root cause of "99.0 yr / РѓгРђћ":** operational measures compute savings from
`annual_consumption_kwh`; residential/data-poor buildings (e.g. B011) have none Рєњ
saving 0 Рєњ payback sentinel `99.0`. Approved approach **Yol B (hybrid)**:

1. **Honest sentinel:** the **10 output payback** `.otherwise(lit(99.0))` Рєњ `NULL`.
   Two of the original twelve `99.0` are deliberately **kept**: the solar
   pre-filter `_est_payback` (used as `< attractive` gate) and the priority-rank
   sort key Рђћ nulling them would break logic. Invariant checked in the patch.
2. **Benchmark estimate (3 quick-wins only Рђћ LED, BMS, HVAC scheduling):** new
   `est_consumption_kwh = coalesce(metered, conditioned_area_m2 ├Ќ _elec_eui_bench)`.
   `_elec_eui_bench` = **electricity-only** EUI midpoint by type (kWh/m┬▓┬иyr):
   residential 28, office 85, retail 150, hotel 90, hospital/health 130, school 40,
   logistics/warehouse 50, industrial 120, default 70. **Electricity-only** on
   purpose Рђћ residential total (~130) is heating-dominated; non-heating electricity
   is ~20РђЊ35 (German destatis/ODYSSEE: ~10 lighting + ~15 appliances; offices
   60РђЊ100 per CIBSE TM46 / NL CBS). Reviewer: Mert. Indicative, not guarantees.
3. **Range, not false precision:** numeric `annual_saving_eur` = midpoint; a
   `┬▒30%` plain-language range + "benchmark estimate (no metered consumption)" is
   appended to `description_en` (post-union, quick-wins only). Commercial buildings
   with metered data are unaffected (estimate flag false Рєњ behaves exactly as before).

**No schema change** to `gold_recommendations` (the estimate flag is dropped at the
final select). py_compile clean; invariant held (exactly 1 surviving `99.0`).
