# Master Proposal Template (LLM uses this as the skeleton)

This is the structural template the LLM follows. It must produce a 5-ingredient
proposal in 220-320 words. Do not exceed 320 words — clients skip long pitches.

## The 5 Required Ingredients

1. **Pain mirror (opening)** — 1-2 sentences in client's language repeating the
   problem they wrote about. Avoid generic "I saw your post" openings.

2. **Relevant EnergyLens proof point** — 1 specific artifact (see
   `personal.yml → proof_artifacts`) tied to their pain. One short paragraph,
   not a CV dump.

3. **Specific question about their brief** — proves the proposal is not
   boilerplate. Phrase as a real question they would answer in reply.

4. **Productized package match** — name the package (P1/P2/P3/P4) with price
   and duration. Or hourly fallback. Always offer 2 options when possible
   (forces choice).

5. **Domain jargon paragraph** — 2 sentences that show CSRD/Fabric literacy.
   Examples: "market-based vs location-based Scope 2", "Direct Lake mode",
   "ESRS E1 datapoints E1-1 to E1-9", "OneLake medallion lineage".

## Sign-off

End with a single CTA sentence + bookable Calendly link.
Closing: "Best, Mert"

## Tone

- Direct, senior, no apologizing
- No "I would love to" or "I think I might be able to"
- Use "will" not "would"
- One paragraph per ingredient, single line breaks between

## Forbidden Patterns

- Do not start with "Hello [name]" — start with the pain
- Do not use the word "passionate"
- Do not list credentials in the first paragraph
- Do not promise specific deliverables not in the package
- Do not use emojis or markdown bold/italic (it renders weirdly across platforms)
- Do not exceed 320 words total

## Output Format

Return ONLY the proposal text. No JSON, no preamble. Plain text, single column,
paragraph breaks between ingredients.
